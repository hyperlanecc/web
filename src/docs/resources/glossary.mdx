## 代理

Hyperlane [代理](/docs/protocol/agents)是读取和写入 Hyperlane 智能合约状态的链下参与者。

示例代理包括[验证器](/docs/protocol/agents/validators)和[中继器](/docs/protocol/agents/relayer)。

## 聚合 ISM

[聚合 ISM](/docs/protocol/ISM/standard-ISMs/aggregation-ISM) 是一种跨链安全模块，通过要求 `n` 个 [ISM](/docs/resources/glossary#interchain-security-module) 中的 `m` 个验证特定跨链消息，从许多 ISM 聚合安全性。

## 检查点

检查点是一个（默克尔根，索引）元组，对应于特定时间点[邮箱](/docs/resources/glossary#mailbox)增量默克尔树的状态。

[验证器](/docs/resources/glossary#validator)的检查点签名用于[多重签名 ISM](/docs/resources/glossary#multisig-ism)。

## 抵押品链

对于 [Hyperlane Warp 路由](/docs/resources/glossary#warp-route)，将[抵押品代币](/docs/resources/glossary#collateral-token)存入 Warp 路由的链。

## 抵押品代币

对于 [Hyperlane Warp 路由](/docs/resources/glossary#warp-route)，存入[抵押品链](/docs/resources/glossary#collateral-chain)以在远程[合成链](/docs/resources/glossary#synthetic-chain)上创建包装代币的代币。

## 默认 ISM

如果消息接收者没有指定自己的 ISM，将用于验证入站消息的[跨链安全模块](/docs/resources/glossary#interchain-security-module)。

## 域

特定链的唯一标识符，Hyperlane 协议用它来确定消息的源和目标。

可能与 EVM 链 ID 相同，但并非总是如此。有关已知 Hyperlane 域 ID 的列表，请参见[域](/docs/reference/domains/)。

## 跨链账户 (ICA)

允许用户进行跨链智能合约调用的[中间件](/docs/resources/glossary#middleware)智能合约。

例如，DAO 可以使用跨链账户拥有远程链上的合约。

有关更多信息，请参见[跨链账户](/docs/applications/interchain-account)。

## 跨链 Gas 支付主

由[中继器](/docs/resources/glossary#relayer)部署的智能合约，接受源链上的支付以在目标链上传递消息。

有关更多信息，请参见[跨链 Gas 支付](/docs/protocol/core/interchain-gas-payment)

## 跨链查询 (IQS)

允许用户进行跨链视图调用的[中间件](/docs/resources/glossary#middleware)智能合约。

例如，智能合约可以使用跨链查询从远程链查找预言机汇率或代币余额。

## 跨链安全模块 (ISM)[​](#interchain-security-module-ism "Direct link to Interchain Security Module (ISM)")

[跨链安全模块](/docs/protocol/ISM/modular-security) (ISM) 是为 Hyperlane 的跨链[邮箱](/docs/protocol/core/mailbox)提供安全性的智能合约。

ISM 负责验证在目标链上传递的跨链消息_确实在_源链上发送。

## 邮箱

可以说是最重要的 Hyperlane 智能合约，[邮箱](/docs/protocol/core/interchain-gas-payment)公开了开发者可用于[发送](/docs/reference/messaging/send)和[接收](/docs/reference/messaging/receive)跨链消息的接口。

## 中间件[​](#middleware "Direct link to Middleware")

发送和接收消息并公开面向开发者的 API 的智能合约。开发者应使用此 API，而不是直接与[邮箱](/docs/resources/glossary#mailbox)交互。

示例中间件包括[跨链账户-ica](/docs/resources/glossary#interchain-accounts-ica)和[跨链查询-iqs](/docs/resources/glossary#interchain-queries-iqs)。

## 多重签名 ISM

[多重签名 ISM](/docs/protocol/ISM/standard-ISMs/multisig-ISM) 是一种[跨链安全模块](/docs/resources/glossary#interchain-security-module)，使用 `n` 个[验证器](/docs/resources/glossary#validator)签名中的 `m` 个来验证特定跨链消息。

## 无需许可互操作性[​](#permissionless-interoperability "Direct link to Permissionless Interoperability")

Hyperlane 可以由任何人部署到任何链，无论是一层、rollup 还是应用链，允许该链无缝通信。Hyperlane 协议的开放设计被称为无需许可互操作性 (PI)。由 Hyperlane 核心团队以外的机构部署的链被称为"PI 链"。

## 中继器

[中继器](/docs/protocol/agents/relayer)是负责将消息从其源链传递到目标链的 Hyperlane [代理](/docs/resources/glossary#agent)。

中继器是不受信任的，任何人都可以操作中继器。

## 路由 ISM

[路由 ISM](/docs/protocol/ISM/standard-ISMs/routing-ISM) 是一种[跨链安全模块](/docs/resources/glossary#interchain-security-module)，根据正在传递的消息委托给不同的 ISM。

例如，路由 ISM 可以根据消息发送的源链使用不同的 ISM。

## 合成链

对于 [Hyperlane Warp 路由](/docs/resources/glossary#warp-route)，铸造包装代币的链。这些包装代币由已在[抵押品链](/docs/resources/glossary#collateral-chain)上锁定的[抵押品代币](/docs/resources/glossary#collateral-token)支持。

## 验证器

[验证器](/docs/protocol/agents/validators)是负责证明从源链发送的消息的 Hyperlane [代理](/docs/resources/glossary#agent)。

[多重签名 ISM](/docs/resources/glossary#multisig-ism) 使用验证器签名为入站消息提供安全性。

## Hyperlane Warp 路由

[Hyperlane Warp 路由](/docs/applications/warp-routes/overview)是 Hyperlane 对代币桥接概念的实现，允许您通过 Hyperlane 无需许可地将任何类似 ERC20 的资产转移到任何链。

阅读更多关于如何[部署 Hyperlane Warp 路由](/docs/guides/quickstart/deploy-warp-route)的信息。
